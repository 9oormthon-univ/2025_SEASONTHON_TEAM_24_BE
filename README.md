# [1] Git 규칙

## (1) 커밋 규칙
* **Fix**: 버그 수정
* **Docs**: 문서 수정
* **Chore**: 빌드, 패키지 관리, 파일 삭제 등 기타 작업 (간단하고 하찮은? 일들)
* **Refactor**: 코드 리팩토링
* **Feat**: 새로운 기능 추가
* **Merge**: 병합 작업

ex) 
Feat: 한글

## (2) 브랜치 관리 전략
**Git Flow**
* main (배포환경)
* develop (개발환경)
* Feat/#이슈번호 (새로운 기능 추가시)

# [2] 이슈 작성 규칙

## (1) 이슈 제목 작성 규칙
* **[Feature]** 새로운 기능 추가
* **[Fix]** 버그 수정 
* **[Documentation]** 문서 관련
* **[Refactor]** 코드 리팩토링
* **[Chore]** 기타 작업
* **[Hotfix]** 긴급 수정
* **[Security]** 보안 관련

ex) 
[Feature] 회원관리
